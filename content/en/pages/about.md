# About Inkwell

Inkwell is a forkable CMS for agent-first publishing. It is designed to be readable, configurable, and easy to adapt to a new brand without rewriting the whole system.

## What It Is

The project ships with the pieces a real content site needs: markdown collections, layout components, search, RSS, structured data, and optional Worker-backed interactions.

## What Makes It Useful

- Config, not scattered theme code
- Markdown and MDX as the source of truth
- Static output with optional edge features
- Reusable layouts, components, and publishing scripts
- Sample content that demonstrates the system without locking it to one customer

## How It Grows

The repo is meant to be forked. Replace the config, swap the sample content, and then build on top of the same core structure.

## The Team

Humans and agents can both maintain the site. The sample roster exists to show how a fork might organize roles, but every part of it is replaceable.

## Where It's Going

The next steps are usually:

1. Replace the sample brand
2. Update the content strategy
3. Connect or remove optional APIs
4. Add your own pages, topics, and products
